## Problem
https://leetcode.com/problems/implement-queue-using-stacks/

## Solution
1. push to s1, pop from s2
2. push as normal
3. pop: move from s1 to s2 when s2 is empty, then pop from s2

## Code
```java
class MyQueue {
    private Stack<Integer> stack1;
    private Stack<Integer> stack2;
    private int front;

    public MyQueue() {
        stack1 = new Stack<Integer>();
        stack2 = new Stack<Integer>();
    }
    
    public void push(int x) {
        if(stack1.isEmpty()) front = x;
        stack1.push(x);
    }
    
    public int pop() {
        if(stack2.isEmpty()){
            while(!stack1.isEmpty()){
                stack2.push(stack1.pop());
            }
        }
        return stack2.pop();       
    }
    
    public int peek() {
        return stack2.isEmpty() ? front : stack2.peek();
    }
    
    public boolean empty() {
        return stack1.isEmpty() && stack2.isEmpty();
    }
}

/**
 * Your MyQueue object will be instantiated and called as such:
 * MyQueue obj = new MyQueue();
 * obj.push(x);
 * int param_2 = obj.pop();
 * int param_3 = obj.peek();
 * boolean param_4 = obj.empty();
 */
```

## Complexity
1. push:

    Time: O(1) 

    Space: O(N) need additional memory to store the queue elements

2. pop:

    Time: O(1+), Amortized O(1), Worst-case O(n). 

    Space: O(1) 

3. peek and isEmpty:

    Time: O(1) 

    Space: O(1)
